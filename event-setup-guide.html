Understanding Series and Performances

In QRTick, all events are organized using a "Series and Performances" model:

Key Concept: A "Series" is the overall event container, while "Performances" are the individual occurrences within that series.

Series vs. Performances

  • Series: The overall event concept or program (e.g., "Summer Concert Series" or "Annual Charity Gala")
  • Performances: Individual occurrences within the series, each with its own date, time, and potentially unique details
  • Single Event: For a one-time event, you'll create a series with just one performance

Examples

  • Multiple Performances: A theater production running for several nights would be a series with multiple performances (one for each night)
  • Single Performance: A one-day charity fundraiser would be a series with a single performance

Event Descriptions

Series Description

The series description provides general information about your overall event:

  • Overall concept and purpose of the event
  • General information that applies to all performances
  • Host organization or venue details
  • Any overarching themes or information

Event Banner

Each event series requires a banner image:

  • Required ratio: 1.2:1 (width:height), such as 600 × 500 pixels
  • Content: Keep it simple - the event name and date range is usually sufficient
  • Design: Since the banner will be accompanied by event information text, avoid cluttering it with too much detail

Performance Descriptions (Optional)

For each performance, you can optionally add a specific description:

  • Unique details for this specific date/time
  • Special guests or features for this performance only
  • Any variations from the standard series format
  • Performance-specific instructions or requirements

Setting Up Ticket Types

Single vs. Multiple Ticket Types

You'll need to decide whether your event requires a single ticket type or multiple types:

Single Ticket Type

  • Simplest approach - all attendees pay the same price
  • Typically labeled as "General Admission"
  • Best for straightforward events with uniform access

Multiple Ticket Types

  • Different pricing tiers based on benefits, timing, or attendee categories
  • Common examples include:
    • Early Bird: Discounted tickets available before a certain date
    • VIP: Premium tickets with additional benefits
    • Standard/Regular: Normal admission tickets
    • Age-Based: Different prices for adults, children, seniors, etc.
    • Group Packages: Discounted rates for multiple tickets
Important: Each ticket type must have a clear, descriptive name that communicates its value to potential attendees.

Pricing Strategy

Setting Base Prices

For each ticket type, you'll need to set a base price:

  • Consider your costs, target audience, and event value
  • Research comparable events in your area
  • Account for QRTick processing fees in your pricing strategy

Performance-Specific Pricing

You can optionally set different prices for specific performances:

  • Higher prices for premium dates (weekends, holidays)
  • Discounted prices for less popular times (weekday matinees)
  • Special promotional pricing for specific performances

Example pricing structure for a concert series:

Ticket Type Base Price Friday Night Performance Saturday Night Performance
Early Bird $25.00 $25.00 $25.00
General Admission $35.00 $35.00 $40.00
VIP $75.00 $75.00 $85.00

Event Setup Checklist

  • Determine if your event is a series with multiple performances or a single event
  • Create a compelling series description that explains your overall event
  • Prepare a banner image with 1.2:1 ratio (e.g., 600 × 500 pixels)
  • For each performance, decide if you need a specific description
  • Determine how many ticket types you need
  • Name each ticket type clearly (e.g., "General Admission", "VIP", "Early Bird")
  • Set base prices for each ticket type
  • Decide if any performances need special pricing
  • Set ticket availability dates and quantities
  • Review all settings before publishing
